입력
▶이렇게 진행합니다
개요 화면 [지침] 영역의 [편집 ✏] 버튼을 누른 뒤, 아래 지침 전문을 복사해 붙여넣고 [저장]합니다.

• 이 에이전트의 역할: 대상 폴더의 파일 목록만 추출하는 리스트 추출 에이전트 — 파일 본문은 읽지 않습니다.
• 핵심: 입력 timestamp(UTC→KST) · 대상 폴더(OneDrive "홍범식 CEO의 파일 - BSH > 1. Daily > 날짜 폴더") · 파일명 파싱 HHMM_N)이름.확장자 · 정해진 [FILES] 출력 형식 · URL 파일은 InternetShortcut의 URL 추가.
📋 복사용 — 지침(Instructions) 전문
# 역할
너는 대상 폴더의 파일 목록을 뽑아 다음 에이전트로 넘기는 리스트 추출 에이전트다.
파일 원문은 절대 읽거나 출력하지 않는다. 오직 "목록(경로·종류)"만 만든다.

# 입력 변수
- timestamp: {{timestamp}} UTC로 조회될테니 KST로 변환.
- 값이 없으면 오늘 날짜로 반환

# 0단계. 대상 폴더 결정
- timestamp 기준 날짜를 "MM.DD(요일)" 형태로 만든다.
· 월/일 1~2자리, 요일은 한글 약식(월·화·수·목·금·토·일)
· 예: 2026-06-25 → "6.25(목)"
- OneDrive 내 홍범식 CEO의 파일 - BSH 폴더 하위의 1. Daily 폴더 하위의 폴더 안의 파일만 가져온다.
- 폴더를 못 찾으면 본문 대신 "해당 날짜 폴더를 찾을 수 없음: <폴더명>"만 출력하고 종료.
- 지워진 파일이 있을 수 있음. 지워진 파일은 대상에서 제외.

# 1단계. 파일명 파싱
- 파일명 규칙: HHMM_N)이름.확장자 (예: 0900_1)ABC.pdf, 0900_2)EDF.url, 1100_1)GHI.pdf)
· HHMM = 보고 시각 (4자리)
· N = 같은 시각 내 순번
· 확장자로 종류 판별: .pdf → PDF, .url → URL
- 규칙에 안 맞는 파일은 목록에 넣되 HHMM/IDX를 "미상"으로 표기한다.

# 출력 형식 (이 형식만, 다른 말 금지)
[FOLDER]: <MM월 DD일(요일)>
[FOLDERID]: <폴더 ID>
[EXTRACTED_AT]: {{timestamp}}
[FILE_COUNT]: <총 파일 수>
[FILES]:
- HHMM=<0900> | IDX=<1> | NAME=<ABC.pdf> | FILEID=<AAABBBBCCCDD12472362> | TYPE=<PDF> | PATH=<전체 경로>
- HHMM=<0900> | IDX=<2> | NAME=<EDF.url> | FILEID=<AAABBBBCCCDD12472362> | TYPE=<URL> | PATH=<전체 경로> | URL=<파일 내 URL경로>
- HHMM=<1100> | IDX=<1> | NAME=<GHI.pdf> | FILEID=<AAABBBBCCCDD12472362> | TYPE=<PDF> | PATH=<전체 경로>

FILE TYPE 이 URL 일 경우 파일 내용의
[InternetShortcut]
URL=
안에 들어있는 URL 경로를 추가로 작성해줘.

# 금지 사항
- 파일 본문을 읽거나 요약하지 마라. 목록 외 어떤 설명·인사말도 출력하지 마라.
- HHMM 순서로 정렬해서 출력한다(오름차순).

  • ✓지침 전문이 입력·저장되고 문자 수(예: 1301/8000)가 표시된 상태

테스트 & 결과 확인

게시한 워크플로를 수동 실행해 두 에이전트가 이어 동작하는 과정을 모니터링하고, Teams로 도착한 요약 브리핑과 이어지는 Follow-up 대화까지 확인합니다.

STEP 30–37테스트 — 흐름 실행 → 모니터링 → Teams 브리핑 → Follow-up
30

흐름 테스트 — 수동 실행

테스트
▶이렇게 진행합니다
게시한 워크플로를 바로 실행해 봅니다.

• 우측 상단 [테스트]를 눌러 [흐름 테스트] 패널을 엽니다.
• [수동]을 선택하고 [테스트]를 누릅니다. (테스트 실행은 용량을 사용하지 않습니다)
✓ 이렇게 되면 성공
  • ✓흐름 테스트(수동) 패널에서 [테스트]를 눌러 실행을 시작한 상태
31

흐름 실행

실행
▶이렇게 진행합니다
[흐름 실행] 패널에서 "이 흐름은 Agents 및 Microsoft Teams을 사용합니다" 안내를 확인하고 [흐름 실행]을 누릅니다.

• "흐름 실행이 시작되었습니다" 안내가 뜨면, [흐름 실행 페이지]로 이동해 실행 상태를 봅니다.
✓ 이렇게 되면 성공
  • ✓워크플로 실행이 시작되고 흐름 실행 페이지 안내가 표시된 상태
32

실행 상태 확인 — 활동 · 세부 흐름도

모니터링
▶이렇게 진행합니다
[활동] 탭의 실행 목록에서 방금 시작된 실행(상태: 실행 중)을 확인합니다.

• [시작 시간]을 클릭하면 세부 워크플로 실행 상태(흐름도)를 볼 수 있습니다.
• 되풀이 → Run CEO Staff GetReports Agent → Run CEO Staff → 채팅 또는 채널에서 메시지 게시 순으로 진행됩니다.
✓ 이렇게 되면 성공
  • ✓실행 세부 흐름도가 표시되고 각 단계 진행 상태를 볼 수 있는 상태
33

1단계 결과 — Run CEO Staff GetReports Agent

확인
▶이렇게 진행합니다
첫 단계 [Run CEO Staff GetReports Agent]가 완료(Succeeded)됩니다. 단계를 클릭하면 디테일을 볼 수 있습니다.

• 안내: "6.26 일자 파일 분석해줘" (날짜 식이 적용된 입력)
• 입력: 에이전트 = CEO Staff GetReports Agent
• 출력: 에이전트 응답 = [FOLDER] … [FILES] … — 해당 날짜 폴더의 파일 목록이 추출됩니다.
✓ 이렇게 되면 성공
  • ✓GetReports 단계가 Succeeded이고 파일 목록([FILES])이 출력된 상태
34

2단계 결과 — Run CEO Staff 요약

확인
▶이렇게 진행합니다
이어서 [Run CEO Staff] 단계가 진행됩니다.

• 디테일의 안내에는 첫 단계가 넘긴 [FOLDER] … [FILES] … 파일 목록이 입력으로 들어옵니다.
• 실행이 완료되면 모든 보고서의 요약본(#제목 / #요약)이 결과로 출력됩니다. (각 파일을 Work IQ로 읽어 3줄 요약)
✓ 이렇게 되면 성공
  • ✓CEO Staff가 6개 파일을 읽어 #제목 / #요약 요약본을 생성한 상태
35

Teams 브리핑 전달 — 채팅 또는 채널에서 메시지 게시

확인
▶이렇게 진행합니다
요약 결과물은 마지막 단계 [채팅 또는 채널에서 메시지 게시](Teams 커넥터)를 통해 Teams로 전달됩니다.

• Parameters: poster=Power Virtual Agents · body/bot=crcce_CEOStaff · recipient=admin@… · messageBody=요약
• Teams에서 "수집된 정보를 바탕으로 6개 파일의 요약" 메시지를 수신합니다. 웹은 물론 모바일에서도 확인할 수 있습니다.
✓ 이렇게 되면 성공
  • ✓요약 브리핑이 Teams로 전달되어 수신된 상태
36

Follow-up 대화 — 컨텍스트 · 메모리

활용
▶이렇게 진행합니다
이어서 보고 자료에 대해 궁금한 점을 바로 질문할 수 있습니다.

• 에이전트는 컨텍스트를 유지하고 메모리를 활용해 매일 더 나아지는 답변을 제공합니다.
• 예) "현대 기아차와 SK가 미래 모빌리티 관련 협업할 수 있는 아이템이 뭘까" → 협업 아이템 분류 표로 답변합니다.
✓ 이렇게 되면 성공
  • ✓Follow-up 질문에 컨텍스트 기반 답변을 받는 상태
37

심화 질문 — AX × Microsoft 협업 전략 · 시나리오 완성

활용
▶이렇게 진행합니다
금요일의 마지막 보고였던 "AX 관련 Microsoft 협업 전략"을 더 상세히 물어봅니다.

• 예) "Microsoft와 전략적 제휴할 수 있는 분야에 대한 아이디어 상세히 알려줘"
• SK 강점 × Microsoft 강점 × 기대 효과를 정리한 전략적 제휴 종합 요약 표로 답변합니다.

→ 두 에이전트 + 주기 실행 워크플로 + Teams 브리핑 + Follow-up까지, CEO 보고 브리핑 시나리오가 완성되었습니다.
✓ 이렇게 되면 성공
  • ✓보고 맥락을 활용한 심화 답변까지 확인 — CEO 보고 브리핑 시나리오 전체 완성
LAB COMPLETE

축하합니다 — CEO 보고 브리핑 시나리오를 완성했습니다

OneDrive 보고 폴더에서 목록을 추출하고 각 파일을 요약하는 두 에이전트를 만들어 게시하고, 정해진 시각에 자동 실행해 Teams로 브리핑한 뒤 Follow-up 질문까지 답하는 전 과정을 완성했습니다. 같은 방식으로 부서별 자동 브리핑을 얼마든지 늘릴 수 있습니다.
